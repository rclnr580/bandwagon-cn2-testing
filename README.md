## 搬瓦工 CN2 是什么：买之前先搞清楚这件事

搬瓦工的 CN2 线路，指的是走中国电信 CN2 骨干网的路由方式。和普通 163 线路（也叫 ChinaNet 或 AS4134）相比，CN2 的核心优势是：晚高峰丢包少、延迟更稳定。

不过"CN2"这个词在搬瓦工体系里还得细分，分成三个层级：

- **CN2 GT（Global Transit）**：也就是 CN2 普通线路。去程走 CN2，但回程不一定保证，高峰期会绕 163，和全程 CN2 体验差不少
- **CN2 GIA（Global Internet Access）**：双向 CN2，去程回程都走 CN2 精品线路，不管什么时段都更稳
- **CN2 GIA-E（E-Commerce）**：在 CN2 GIA 基础上，带宽提升到 2.5Gbps，并且支持 12+ 个机房随意切换——这是目前搬瓦工性价比最高的线路

## 搬瓦工 CN2 测试：三种方法，从简单到详细

### 方法一：直接 Ping 测试 IP（最快）

最基础的方式。拿到各机房的测试 IP 地址，直接打开终端 ping 一下，看延迟数字。

Windows 用户：打开 PowerShell 或者命令行，输入 `ping IP地址`

macOS / Linux 用户：直接打开终端，`ping IP地址`

延迟数字的大概参考：

- **50ms 以下**：非常快，亚洲机房（香港、日本）才能到这个范围
- **150-200ms**：美西 CN2 GIA 的正常范围，日常使用可以接受
- **200ms 以上**：美东机房，距离更远，延迟相应更高

这个方法的局限是只能看瞬时延迟，看不出丢包，也看不出晚高峰的真实表现。

### 方法二：全国测速工具

如果你想知道的是全国各地到某个机房的平均延迟，而不只是自己所在城市的延迟，可以去 `ping.chinaz.com` 输入测试 IP，它会自动调用全国各节点帮你测一遍，出一张延迟热力图。

实际用起来很直观——你能看到华北、华南、华东、西南各地延迟的差异，也能看出哪些地区对这个机房特别友好。

### 方法三：SpeedTest 测速（看带宽）

上面两个方法测的是延迟，这个方法测的是实际带宽。搬瓦工各主要机房都有官方或演示站点内嵌的 SpeedTest 页面，可以直接测上传下载速度。

和 ping 不同，SpeedTest 对时段敏感。晚上 9-11 点的结果和早上 9 点的结果差别会很明显——高峰期带宽被压缩是正常现象，CN2 GIA 线路的优势就在这时候体现出来。

## 测完延迟之后，路由追踪才是关键

Ping 看到的延迟数字只是结果，路由追踪才能告诉你数据包走的是哪条路。

如果你的延迟偏高，但路由追踪显示中国境内就接入了 CN2 节点（路由里看到 `59.43.x.x` 的 IP 段），那说明线路本身没问题，延迟是正常的物理距离决定的。

如果路由追踪显示在国内走了一大圈 163 线路才出境，那延迟高的原因就是线路质量问题，不是 CN2 GIA 该有的表现。

路由追踪工具：Windows 用 `tracert IP`，macOS / Linux 用 `traceroute IP`，或者在线工具里输 IP 也可以。
